GIT

  • The code housed in GitHub is organized into repositories
  • When working on code, we don't work directly on the main "branch" of the repository. Instead, we create our own branch, in which we can work without getting in our teammates' way. This branching structure enables teams to be working on several features at once without conflicting.
  • Here are some often-used git commands entered on the command line:
    • git status: checks what branch we are currently on
    • git checkout -b [branch-name]: creates a new branch and switches to it
    • git add -A: stages all the edited (or newly added) files to be ready for a commit
    • git commit -m "[commit notes]": commits all staged files to the current branch, and includes the given notes
    • git pull origin main: pulls down any changes that have been made to the main branch, and updates your working branch to reflect them
    • git push origin [branch-name]: pushes your updated local copy of your working branch up to GitHub
  • After our work on a branch is complete, we file a "Pull Request". A Pull request is a process whereby your code changes can be viewed by other team members and approved, before being merged into the main branch.
